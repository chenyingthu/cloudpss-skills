# 系统安全评估场景

## US-021: N-1预想事故扫描 🟢

### 用户角色
安全分析工程师，需要进行常规N-1扫描

### 目标
对系统进行全面的N-1预想事故扫描

### 场景描述
用户需要对当前运行方式进行N-1安全评估。
1. 定义扫描范围（线路、变压器、发电机）
2. 执行N-1扫描
3. 输出结果：
   - 严重故障排序
   - 越限元件统计
   - 薄弱环节识别
4. 生成N-1扫描报告

### 技能要求
- `n1Analysis.runFullScan()` - 完整N-1扫描
- `n1Analysis.analyzeWeaknesses()` - 薄弱环节分析
- `n1Analysis.generateReport()` - 报告生成

### 验收标准
- [ ] 扫描覆盖所有指定元件
- [ ] 严重程度评估准确
- [ ] 报告内容完整

### 优先级
P0

3. 执行N-1扫描（考虑检修状态）

### 验收标准
- [ ] 检修状态设置正确

- [ ] 结论明确

## US-028: 稳定裕度评估 🟡

### 用户角色
稳定分析工程师，需要评估系统稳定裕度

### 目标
评估系统的静态电压稳定裕度

### 场景描述
用户需要知道系统距离电压崩溃还有多少裕度。
1. 确定负荷增长方向
2. 逐步增加负荷，计算潮流
3. 识别临界点（潮流不收敛点）
4. 计算稳定裕度指标：
   - 负荷裕度
   - 关键节点识别
   - 薄弱区域

### 技能要求
- `stability.voltageStabilityMargin()` - 电压稳定裕度（需新增）
- `batchEnhanced.loadGrowthScan()` - 负荷增长扫描

### 验收标准
- [ ] 临界点识别准确
- [ ] 裕度计算合理
- [ ] 薄弱区域识别正确

### 优先级
P2
