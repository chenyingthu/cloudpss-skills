# 批量计算与优化场景

## US-029: 年度方式批量计算 🔴

### 用户角色
方式计划工程师，需要编制年度运行方式

### 目标
批量计算全年各典型方式的潮流

### 场景描述
用户需要为年度运行方式编制准备基础数据。
1. 定义典型方式列表（丰水期、枯水期、高峰、低谷等）
2. 批量执行潮流计算
3. 汇总分析结果：
   - 各方式潮流分布
   - 越限情况汇总
   - 设备负载统计
4. 生成年度方式计算报告

### 技能要求
- `batchEnhanced.runPowerFlowBatch()` - 批量潮流
- `batchEnhanced.aggregateResults()` - 结果汇总
- `report.generateAnnualReport()` - 年度报告（需新增）

### 验收标准
- [ ] 所有方式计算完成
- [ ] 结果汇总完整
- [ ] 报告格式规范

### 优先级
P1

## US-030: 负荷增长极限分析 🟢

### 用户角色
规划工程师，需要分析系统承载能力

### 目标
确定系统在不违反安全约束下的最大负荷承载能力

### 场景描述
用户需要知道当前电网还能增加多少负荷。
1. 从当前负荷水平开始
2. 逐步增加负荷（步长可调）
3. 每步检查越限情况
4. 确定临界负荷水平
5. 输出分析结果：
   - 最大负荷水平
   - 限制因素（哪条线路/节点先越限）
   - 瓶颈设备列表

### 技能要求
- `batchEnhanced.loadGrowthScan()` - 负荷增长扫描
- `powerFlow.checkViolations()` - 越限检查

### 验收标准
- [ ] 临界点识别准确
- [ ] 瓶颈设备定位正确
- [ ] 分析结果可信

### 优先级
P1

### 验收标准

## US-033: 网损优化分析 🔴

### 用户角色
节能专责，需要优化系统网损

### 目标
分析并优化系统网损

### 场景描述
用户希望降低系统网损，提高运行经济性。
1. 计算当前网损
2. 分析网损分布（线路损耗排名）
3. 模拟降损措施：
   - 无功补偿优化
   - 运行方式调整
   - 变压器档位优化
4. 评估各措施降损效果
5. 推荐最优降损方案

### 技能要求
- `loss.analyze()` - 网损分析（需新增）
- `loss.optimize()` - 网损优化（需新增）
- `powerFlow.getBranchFlows()` - 支路功率

### 验收标准
- [ ] 网损计算准确
- [ ] 降损措施有效
- [ ] 方案经济性分析合理

### 优先级
P2

## US-035: 潮流收敛性诊断 🟡

### 用户角色
仿真工程师，遇到潮流不收敛问题

### 目标
诊断潮流不收敛原因并提出解决方案

### 场景描述
用户进行潮流计算时遇到不收敛问题。
1. 执行潮流计算，检测到不收敛
2. 自动进行诊断：
   - 检查数据合理性
   - 识别可能的孤立节点
   - 检查PV/PQ节点设置
3. 输出诊断报告和建议
4. 尝试自动修复（如调整初始值）

### 技能要求
- `diagnosis.checkConvergence()` - 收敛性诊断（需新增）
- `diagnosis.suggestFixes()` - 修复建议（需新增）

### 验收标准
- [ ] 诊断准确率高
- [ ] 建议切实可行
- [ ] 自动修复成功率

### 优先级
P1

   - 经济指标对比表

### 技能要求

- `report.generateComparisonReport()` - 比选报告（需新增）

### 验收标准

- [ ] 对比分析全面
