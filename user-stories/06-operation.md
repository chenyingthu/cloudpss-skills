# 设备运维支持场景

## US-043: 设备参数查询 🟢

### 用户角色
运维工程师，需要查询设备铭牌参数

### 目标
快速查询算例中某设备的详细参数

### 场景描述
用户需要查询某台变压器的详细参数，用于现场运维参考。
1. 在算例中搜索设备（按名称或ID）
2. 查看设备详细信息：
   - 铭牌参数
   - 连接拓扑
   - 当前运行状态（如有仿真结果）
3. 导出设备参数卡片

### 技能要求
- `manage.getModel()` - 获取算例
- `component.getDetails()` - 设备详情（需增强）
- `export.componentCard()` - 参数卡片（需新增）

### 验收标准
- [ ] 设备检索方便
- [ ] 参数信息完整
- [ ] 导出格式规范

### 优先级
P1

## US-045: 线路负载率统计 🟢

### 用户角色
运维专责，需要了解线路负载情况

### 目标
统计各线路的负载率分布

### 场景描述
用户需要了解系统各线路的负载情况，识别重载线路。
1. 运行潮流计算
2. 提取线路负载率
3. 统计分析：
   - 负载率分布
   - 重载线路列表（>80%）
   - 过载线路列表（>100%）
4. 生成线路负载统计报告

### 技能要求
- `powerFlow.getBranchFlows()` - 支路功率
- `statistics.branchLoading()` - 负载统计（需新增）

### 验收标准
- [ ] 统计准确
- [ ] 重载识别正确
- [ ] 报告清晰

### 优先级
P1

## US-046: 设备过载预警 🟡

### 用户角色
监控工程师，需要识别设备过载风险

### 目标
识别并预警设备过载情况

### 场景描述
用户需要知道哪些设备存在过载风险。
1. 运行潮流计算
2. 检查设备越限情况
3. 按严重程度排序
4. 生成预警列表：
   - 严重过载（紧急处理）
   - 一般过载（需关注）
   - 接近限值（提前预警）

### 技能要求
- `powerFlow.checkViolations()` - 越限检查
- `alert.generateWarnings()` - 预警生成（需新增）

### 验收标准
- [ ] 过载识别准确
- [ ] 分级合理
- [ ] 预警及时

### 优先级
P1

## US-047: 检修影响评估 🟡

### 用户角色
检修计划专责，需要评估检修影响

### 目标
评估设备检修对系统的影响

### 场景描述
用户计划对某条线路进行检修，需要评估影响。
1. 模拟设备停运
2. 执行潮流计算
3. 进行N-1扫描（考虑检修状态）
4. 评估影响：
   - 潮流变化
   - 新的风险点
   - 供电可靠性影响
5. 生成检修影响评估报告

### 技能要求
- `manage.setMaintenance()` - 检修设置（需新增）
- `n1Analysis.runFullScan()` - N-1扫描
- `report.maintenanceImpact()` - 影响报告（需新增）

### 验收标准
- [ ] 模拟准确
- [ ] 评估全面
- [ ] 报告实用

### 优先级
P1
